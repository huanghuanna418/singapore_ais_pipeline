# Singapore Harbour Craft AIS Pipeline

A complete pipeline for fetching AIS data, building Continuous Reports
in SeaCabbie format, and estimating electric harbour craft (eHC) energy
consumption — for any Singapore harbour craft operator.

This is the deliverable for tasks 1, 2, 3, 5, and 6 from your project brief.

---

## What it does (in plain English)

1. **Records live AIS data** — listens to ship broadcasts in Singapore
   port and saves them to disk, day by day, for as long as you let it
   run.

2. **Maps vessel names to AIS IDs** — when you give it a Trip Report
   that lists vessels by name, it figures out which AIS broadcasts
   belong to those vessels.

3. **Builds Continuous Reports** — combines a Trip Report with the
   recorded AIS data to produce a Continuous Report in the *exact*
   same column format as SeaCabbie's. So Estee's Flexsim code can use
   it without changes.

4. **Estimates energy use** — for each trip, calculates how many
   kilowatt-hours an electric boat would have used, based on the
   speed profile and a chosen eHC model.

5. **Has a web interface** — a small website that runs on your computer
   so you can do all of the above by clicking buttons instead of
   typing commands.

## Setup (one-time)

1. **Install Python 3.9 or newer** if you don't have it. (You probably
   do — try typing `python --version` in a terminal.)

2. **Install the libraries this project uses.**
   Open a terminal in the project folder and run:

   ```
   pip install pandas numpy openpyxl websockets flask
   ```

3. **Get a free aisstream.io API key.**
   Sign up at https://aisstream.io/ — it takes about a minute.

4. **Tell the project about your key.**
   You have two choices:

   *Option A* — open `config.py` in a text editor and replace
   `PASTE_YOUR_KEY_HERE` with your key.

   *Option B* — set an environment variable. In a terminal:
   ```
   export AISSTREAM_API_KEY=your_key_goes_here
   ```
   (You'll need to run this every time you open a new terminal.)

## Real run: the typical workflow

### Workflow A — Using the web interface (easier)

```
python 05_web_ui.py
```

Open http://localhost:5000 in your browser. You'll see a list of
operators. Click on one to:

- Upload its Trip Report
- Resolve MMSIs
- Build the Continuous Report
- Estimate energy
- Download any of the generated files
- See vessel routes on a map

### Workflow B — Using the command line (more control)

```bash
# 1. Drop the operator's Trip Report into operators/<key>/trip_report.xlsx
#    Example: operators/yorklaunch/trip_report.xlsx

# 2. Resolve the vessel names to MMSIs
python 01_resolve_mmsi.py --operator yorklaunch
# (For any vessel still missing an MMSI, look it up on vesselfinder.com,
#  open operators/yorklaunch/vessel_mmsi.csv in Excel, type the number,
#  save, and re-run step 1.)

# 3. Start recording AIS. Leave this running.
#    For one operator:
python 02_record_ais.py --operator yorklaunch
#    For everyone:
python 02_record_ais.py --operator all

# 4. After enough data is collected, build the Continuous Report
python 03_build_continuous_report.py --operator yorklaunch

# 5. Estimate energy
python 04_energy_consumption.py --operator yorklaunch

# 6. Verify the Continuous Report can be loaded by Estee's Flexsim code
python 06_verify_flexsim_compat.py --operator yorklaunch
```

## Important: the historical data problem

aisstream.io is a LIVE service. It can't give you yesterday's data.
To collect one month of data we need to run the recorder for one month.

**Realistic options:**

1. **Run the recorder continuously on a machine that's always on**
   (a lab desktop, a cheap VPS, anything that doesn't sleep). Even
   without operator filters, you can record everything in Singapore
   waters now and filter later.

2. **Buy historical data** from Spire, MarineTraffic, or ExactAIS.
   Costs real money. Talk to your prof first.

3. **Ask if SIT has an MPA data-sharing agreement.** Government
   sources have years of archives.

## Important: the eHC specifications problem

`04_energy_consumption.py` contains placeholder eHC specs (max power,
max speed, hotel load, battery capacity). These are **made up**.

When your prof gives you real eHC specs (from MPA's reference designs
or from a vendor), edit `EHC_MODELS` near the top of
`04_energy_consumption.py` and replace the placeholder numbers.

The energy model itself is also a simplification — it uses a cube-of-
speed drag formula plus a constant hotel load. That's a reasonable
first approximation, but real eHC modelling would account for hull
shape, payload, sea state, motor efficiency, and battery
characteristics. Good enough for ranking trips by energy intensity;
not good enough for a final battery-sizing decision.

---

## Important: time zones

- **Trip Reports** use Singapore local time (UTC+8).
- **AIS data** uses UTC.
- The Continuous Report builder converts UTC → SGT before matching, so
  you don't have to worry about it.

If your operator's Trip Report uses a different time zone, edit
`load_archive_for_window` in `03_build_continuous_report.py`.

## Troubleshooting

**"AISSTREAM_API_KEY not set"** — see Setup step 4.

**"Connection lost / timed out during opening handshake"** — your
internet is too slow for a websocket connection. The recorder
auto-reconnects, so usually fine to ignore. If it keeps failing for
hours, switch networks.

**"No AIS data covers the trip report window"** — you haven't recorded
enough AIS data yet, OR the trip dates are older than your archive.
Run the recorder longer.

**"Vessel still needs an MMSI"** — open `operators/<key>/vessel_mmsi.csv`
in Excel, look up the missing MMSIs on vesselfinder.com, type them in,
save, re-run step 1.

**"No trips with enough data to estimate energy"** — the Continuous
Report has fewer than 2 pings per trip. Probably means the recorder
hadn't started yet when the trips happened.
